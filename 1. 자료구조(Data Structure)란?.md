컴퓨터가 수많은 데이터를 효율적으로 정리하고 조직화하는 여러가지 구조

### 자료구조의 분류
[그림]
- **단순 자료구조**: 정수, 실수, 문자, 등 기본 제공 구조
- **복합 자료구조**: 여러가지 자료(데이터)가 복합적으로 구성된 구조
	- **선형 자료구조(linear data structure)**: 데이터가 순서대로 나열됨 
		접근 방법
		- **순서 접근(Sequential Access)**
			ex) 
			- 연결 리스트(Linked List): 시작 노드부터 하나씩 다음 노드로 이동하면서 원하는 요소 찾기
		- **직접 접근(Direct Access)**
			ex) 
			- 배열(Array): index i를 사용해서 배열의 i번째 요소 A[i]에 접근 가능
			- 스택(Stack), 큐(Queue), 덱:  접근이 맨 앞과 맨 뒤로 제한됨 
	- **비선형 자료구조(nonlinear ")**: 자료 간 선형적 순서 없이, 더 복잡한 연결을 갖는 형태 
		ex)
		- **트리**: 계층구조 표현에 적합 
			- 조직도, 컴퓨터 폴더
			- 이진 탐색 트리: 탐색에 적합, AVL 트리: 효율성 높인 이진탐색트리 
		- **그래프**: 가장 복잡한 형태의 구조 
			- 지하철 노선도, SNS 인맥 지도, 인터넷 망 등
			- node, edge 로 구성 
			- 가중치 그래프 / 방향 그래프/ 무방향 그래프



### 자료구조의 활용
- 정렬(Sort)
	- 데이터를 어떤 기준을 바탕으로 순서대로 나열, 효율 중요
- 탐색(Search)
	- 특정 데이터 찾기, 적절한 자료 구조와 효율적인 탐색 알고리즘 

### 프로그램 = 자료구조 + 알고리즘 
- 프로그램: 데이터 처리 
- 자료구조: 데이터(자료) 표현 방법 
- 알고리즘: 문제 해결 처리 과정(을 컴퓨터가 이해할 수 있는 언어로 표현한것 - 명령어의 집합)
	- 알고리즘의 조건 
		1. Input: 0개 이상의 입력 존재(없어도 됨)
		2. Output: 1개 이상의 출력 존재
		3. 명백성: 각 명령어의 의미는 명확해야함(모호X)
		4. 유한성: 한정된 수의 명령어 수행 후, 반드시 종료
		5. 유효성: 각 명령어는 실행 가능한 연산 (/0 안됨)
	- 알고리즘 표현 방법
		1. 자연어: 영어나 한국어 등
		2. 흐름도(Flowchart)
		3. Pseudo-Code
			- 대입연산자는 <- 
		4. 특정 프로그래밍 언어
효율적인 자료구조 선정 -> 자료구조에 적합한 알고리즘 선정 


### 추상 자료형
- **추상화(Abstraction)**
	: 복잡한 데이터, 모듈, 시스템 등으로부터 **핵심적인 개념이나 기능**을 간추려 냄 
		불필요한 세부 정보는 삭제-정보은닉기법(information hiding)
- **추상 자료형 (ADT: Abstract Data Type)**
	: 추상적으로 정의한 자료형, 
		What O, How X 
		자료의 집합 / 자료에 가해지는 연산들의 집합에 대한 수학적인 명세
	- 정의 
		- **데이터**: Set(집합)
		- **연산**: 수행 기능 
	- 구현 
		-  **정보은닉**: 구현 세부사항은 외부에서 모르게하고, 
		  외부에는 사용자에게 간단한 **인터페이스(interface)** 만 제공 
		- C++의 경우, 추상 자료형 == 객체 지향, Class 사용  
			- 클래스의 attribute = 데이터
			- 클래스의 method = 연산 
			- 상속
